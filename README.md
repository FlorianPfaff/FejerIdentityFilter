# Positive-Kernel Identity Filter

This repository contains a PyRecEst-style implementation of an identity Fourier filter with positivity-oriented coefficient reduction for circular and hypertoroidal Bayesian filtering.

The package is designed as a small extension around PyRecEst's `HypertoroidalFourierDistribution` and `HypertoroidalFourierFilter` classes. It adds:

- `FejerHypertoroidalFourierDistribution`, an identity Fourier distribution whose coefficient reduction can use sharp truncation, Fejer/Cesaro weights, or Fejer-Korovkin weights.
- `FejerIdentityFilter`, a hypertoroidal identity Fourier filter that defaults to an adaptive Fejer-Korovkin nonnegativity safeguard.
- `examples/compare_with_fourier_filters.py`, a reproducible comparison against PyRecEst's ordinary Fourier identity filter (IFF) and Fourier square-root filter (SqFF).

The repository name still uses "Fejer" because the first prototype implemented plain Fejer means. The recommended variant is now the adaptive Fejer-Korovkin safeguard.

## Motivation

The ordinary IFF represents the density directly by Fourier coefficients. This makes the additive-noise identity prediction step especially simple: the topology-aware convolution is a Hadamard product of identity coefficients. However, multiplication followed by sharp coefficient truncation can introduce negative density values.

The SqFF represents the square root of the density and therefore keeps the reconstructed density nonnegative. The cost is that the prediction step is more complicated because square roots do not commute with convolution.

A positive-kernel identity filter keeps the IFF representation and replaces, or conditionally augments, sharp coefficient truncation after pointwise multiplication by a positive-kernel reduction. The implemented kernels are:

```text
plain Fejer:        lambda_k = 1 - |k| / (K + 1)
Fejer-Korovkin:     lambda_1 = cos(pi / (K + 2))
```

For Fejer-Korovkin, all multipliers are generated by the autocorrelation sequence of the normalized sine vector. This makes the associated trigonometric kernel nonnegative, while damping the first Fourier mode only at second order in `K`.

For a hypertorus, the implementation uses tensor-product weights. For example,

```text
lambda_k = prod_j lambda_{k_j}^{(j)}.
```

## Recommended use

The default filter is adaptive:

```python
import pyrecest.backend as backend
from pyrecest.distributions import WrappedNormalDistribution

from fejer_identity_filter import FejerIdentityFilter

n_coefficients = (33,)
filter_ = FejerIdentityFilter(n_coefficients)

process_noise = WrappedNormalDistribution(backend.array(0.0), backend.array(0.25))
measurement_noise = WrappedNormalDistribution(backend.array(0.0), backend.array(0.20))

filter_.predict_identity(process_noise)
filter_.update_identity(measurement_noise, backend.array([1.0]))
estimate = filter_.filter_state.mean_direction()
```

This first performs the ordinary sharp identity reduction. If the reconstructed density is nonnegative on the diagnostic FFT grid, no smoothing is applied. If grid negativity appears, Fejer-Korovkin damping is activated. The adaptive blend searches for the smallest exponent of the kernel weights that clears the diagnostic grid.

To run the old unconditional Fejer prototype:

```python
filter_ = FejerIdentityFilter(
    (33,),
    reduction_kernel="fejer",
    adaptive_reduction=False,
)
```

To run unconditional Fejer-Korovkin reduction:

```python
filter_ = FejerIdentityFilter(
    (33,),
    reduction_kernel="korovkin",
    adaptive_reduction=False,
)
```

## Comparison with Fourier filters

Run the included comparison script from the repository root:

```bash
python examples/compare_with_fourier_filters.py --coefficients 5 9 15 25 33
```

The script compares:

- PyRecEst IFF: `HypertoroidalFourierFilter(..., transformation="identity")`
- Plain unconditional Fejer identity reduction
- Adaptive Fejer-Korovkin identity safeguard
- PyRecEst SqFF: `HypertoroidalFourierFilter(..., transformation="sqrt")`

It reports runtime per predict/update cycle, circular mean error against a dense-grid reference, integrated absolute density error, minimum PDF value on a diagnostic grid, and negative grid mass.

## Interpretation

This code should not be interpreted as an attempt to beat SqFF on smooth problems where the square root of the density is also smooth. In those cases, SqFF can converge spectrally, whereas any unconditional linear positivity-preserving convolution reduction has a second-order ceiling at best. Plain Fejer is weaker still and has first-order low-frequency damping.

The useful object is instead:

```text
IFF + adaptive positive-kernel safeguard
```

This is intended for pipelines where the identity representation is structurally attractive: cheap repeated predictions, nonlinear prediction via transition tensors, marginalization, mixture operations, and situations where negative truncation artifacts are rare but unacceptable.

## Limitations

Positive-kernel reduction is positivity-preserving when applied as a full convolution reduction to a nonnegative trigonometric polynomial. Adaptive intermediate exponents are a practical grid-level safeguard, not a global analytic certificate. For exact global certificates, one would need a nonnegative projection, sum-of-squares certificate, or Fejer-Riesz/spectral-factor representation.
